    • OpenGL Note: 深度测试

    深度测试默认关闭，需要通过 glEnable(GL_DEPTH_TEST) 开启。

    需要在每次渲染前使用 glClear(GL_COLOR_BUFFER_BIT | GL_DEPTH_BUFFER_BIT 清除深度缓冲区，避免上次渲染的深度缓冲区值残留。

    深度测试开启时，Opengl会测试片段的深度值和深度缓冲区的深度值。如果测试通过，深度缓冲区的值会被更新成新的深度值；如果测试失败，该片段会被丢弃。
    当有场景需要在片段上做深度测试后不更新深度缓冲区的值时，可以通过glDepthMask(GL_FALSE) 禁用，此时深度缓冲区是只读的。

    可以通过glDepthFunc(GL_LESS) 修改深度测试的比较操作，可选的值有：

    • GL_ALWAYS
    • GL_NEVER
    • GL_LESS
    • GL_EQUAL
    • GL_LEQUAL
    • GL_GREATER
    • GL_NOTEQUAL
    • GL_GEQUAL

    深度值计算：屏幕空间的深度缓冲区的值是非线性的

    • 线性深度缓冲区：(z - near) / (far - near)
    • 非线性深度缓冲区： (1/z - 1/near) / (1/far - 1/near)

    可以通过gl_FragCoord的值实现对深度缓冲的可视化。gl_FragCoord的x和y分量表示在屏幕空间的坐标值，(0,0)为左下角，z分量表示片段的深度值。
    FragColor = vec4(vec3(gl_FragCoord.z), 1.0);

    深度冲突：深度缓冲区没有足够的精度决定两个片段的显示。在更远处会有更明显的表示，原因是z值越大，深度缓冲区的精度越小。

    防止深度冲突：

    1. 不要把多个物体摆得太靠近
    2. 尽可能把近平面设置的远一些（精度）
    3. 使用更高精度的深度缓冲（牺牲性能）
